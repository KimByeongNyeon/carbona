# Prompt Archive

작업이 끝나면 사용자가 요청한 내용을 이 폴더에 작업 기록 형태로 정리한다.

## 작성 기준

- 파일명은 `YYYY-MM-DD-작업주제.md` 형식을 사용한다.
- 사용자의 요청을 그대로 옮기지 않고, 의도와 맥락이 드러나도록 자연스럽게 정리한다.
- 짧은 승인, 오타, 구어체 표현은 작업 지시서처럼 읽히도록 다듬는다.
- 작업 의도와 범위를 명확하게 정리한다.
- 실제 변경 파일과 결과를 함께 기록한다.
- 후속 작업이 필요하면 별도로 남긴다.

## 템플릿

```md
# 작업 주제

## 정리된 프롬프트

사용자의 요청을 작업 지시서 형태로 자연스럽게 정리한다.

## 작업 의도

- 요청의 핵심 의도
- 작업 범위

## 변경 내용

- 변경한 파일
- 주요 구현 내용

## 결과

- 검증 결과
- 남은 이슈 또는 후속 작업
```
