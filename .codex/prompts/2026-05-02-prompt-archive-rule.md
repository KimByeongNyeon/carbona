# 프롬프트 아카이브 규칙 정비

## 정리된 프롬프트

프로젝트 작업이 끝날 때마다 사용자의 요청을 `.codex/prompts/` 아래에 보기 좋은 작업 기록으로 남긴다. 기록은 사용자의 문장을 그대로 옮기는 방식이 아니라, 요청 의도와 작업 맥락이 잘 드러나도록 자연스럽게 정리한다.

## 작업 의도

- 작업 종료 후 프롬프트와 작업 내용을 추적할 수 있는 기록 체계를 만든다.
- 구어체 요청이나 짧은 승인 메시지를 그대로 보존하지 않고, 나중에 읽기 쉬운 작업 지시서 형태로 다듬는다.
- 이 방식을 프로젝트 규칙에 반영해 이후 작업에서도 일관되게 적용한다.

## 작업 범위

- `.codex/prompts/` 폴더 생성
- 프롬프트 기록 기준 문서 작성
- `.codex/rules.md`에 프롬프트 기록 규칙 추가 및 문구 조정
