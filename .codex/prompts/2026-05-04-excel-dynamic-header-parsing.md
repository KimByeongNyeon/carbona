# Excel 동적 헤더 파싱 보정

## 정리된 프롬프트

실제 과제 Excel 파일에 제목과 안내문, 오른쪽 참고용 배출계수 영역이 포함되어 있어 기존 미리보기 파서가 헤더를 잘못 인식하는 문제를 수정한다. 고정 행 번호에 의존하지 않고 필수 활동 데이터 헤더를 찾아 그 이후 행만 미리보기 대상으로 파싱한다.

## 작업 의도

- 상단 설명 행이 있는 Excel 파일도 별도 가공 없이 업로드할 수 있게 한다.
- 오른쪽 참고용 배출계수 영역은 활동 데이터로 오인하지 않도록 제외한다.
- `일자(원본)`, `활동 유형`, `설명`, `량`, `단위` 헤더 위치를 기준으로 실제 데이터 행만 추출한다.

## 작업 범위

- Excel sheet를 객체가 아닌 2차원 배열로 읽도록 변경
- 필수 헤더 행 탐색 로직 추가
- 헤더별 column index map 생성
- 헤더 다음 행부터 활동 데이터 컬럼만 매핑
- 빈 행 제외 처리
- 실제 과제 파일의 데이터 구조 확인

## 변경한 파일

- `app/domain/excel/services/excelPreview.service.ts`

## 검증 결과

- Excel 관련 파일 ESLint 통과
- `./node_modules/.bin/tsc --noEmit --pretty false` 통과
- `./node_modules/.bin/next build --webpack` 통과
- 실제 파일 기준 활동 데이터 30행, 활동 유형 `전기`, `원소재`, `운송` 추출 확인

## 후속 작업

- DB 배출계수 이름이 실제 Excel 설명 값인 `한국전력`, `플라스틱 1`, `플라스틱 2`, `트럭`과 일치하는지 확인한다.
- 필요하면 Excel 설명 값과 DB 배출계수 이름 사이의 alias 매핑을 추가한다.
